== Overview

These basic labs introduce OpenShift’s secure-by-default posture. Align workloads to built-in guardrails instead of bypassing them.

Topics covered:

* Container security fundamentals (projects and namespaces)
* Default OpenShift platform security layers
* Basic RBAC
* Workload guardrails (SCC/PSA, quotas, limits)
* Least privilege (non-root, minimal capabilities)
* NetworkPolicies
* Secrets basics
* Trusted images and vulnerability awareness
* Encryption in transit and at rest
* Audit and cluster logs
* Security extension operators (awareness)

Audience: any of the five framework roles. Prefer a xref:index.adoc#choose-your-role[role track] so you only take the labs your role owns (for example, Application Developer skips audit logging and compliance add-ons).
