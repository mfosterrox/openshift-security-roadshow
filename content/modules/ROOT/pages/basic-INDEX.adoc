== Overview

These basic labs introduce OpenShift’s secure-by-default posture. Align workloads to built-in guardrails instead of bypassing them.

Canonical Basic skills covered:

* *B0* — Container security fundamentals (shared via projects/namespaces labs)
* *B1* — Default OpenShift platform security layers — 101-01
* *B2* — Basic RBAC — 101-03
* *B3* — Workload guardrails (SCC/PSA, quotas, limits) — 101-02, 101-04
* *B4* — Least privilege (non-root, minimal capabilities) — 101-01, 101-04, 101-11
* *B5* — NetworkPolicies — 101-05
* *B6* — Secrets basics — 101-06
* *B7* — Trusted images / vulnerability awareness — 101-07, 101-11
* *B8* — Encryption in transit and at rest — 101-08, 101-12
* *B9* — Audit and cluster logs — 101-09
* *B10* — Security extension operators (awareness) — 101-10

Audience: any of the five framework roles. Prefer a xref:index.adoc#choose-your-role[role track] so you only take the labs your role owns (for example, Application Developer skips B9/B10).
