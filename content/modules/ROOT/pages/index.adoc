This course offers hands-on experience with Kubernetes-native security on Red Hat^(R)^ OpenShift Container Platform—covering platform foundations, Advanced Cluster Security (RHACS), and Trusted Software Supply Chain (TSSC). Each module ends with a practical *task* so you apply what you learned before moving on.

== How this Roadshow works

* *This page is home* — Credentials and console links live here. Use the home button anytime you get lost.
* *Navigate with the TOC* — Modules are in the left table of contents, or use *Next* at the bottom of each page.
* *Open consoles in a new tab* — Keep this guide open. On a Mac, use *<cmd> click*; on Windows/Linux, use *<ctrl> click* or right-click → "Open in new tab."
* *Use the bastion* — Run `oc`, `roxctl`, and `podman` on the RHEL bastion (SSH details below).
* *Pick a role* — Choose the track that matches your job. Each track lists only the labs that role owns in the https://ralvares.github.io/openshift-security-framework/[OpenShift Security Skills Framework^].

== Choose your role

Start here. Open the track for your role, then follow the recommended sequence (Basic → Intermediate → Advanced).

* xref:track-application-developer.adoc[Application Developer] — Build and ship secure workloads: least privilege, secrets, trusted images, and baseline pipeline hygiene.
* xref:track-platform-operator.adoc[Platform Operator] — Operate and harden clusters: SCCs, quotas, network baselines, audit, compliance, and day-2 guardrails.
* xref:track-devsecops-engineer.adoc[DevSecOps Engineer] — Embed security in delivery: CI/CD gates, signing, admission policy, runtime feedback, and supply chain controls.
* xref:track-security-architect.adoc[Security Architect] — Design governance and zero-trust strategy across teams, clusters, identity, and exceptions.
* xref:track-network-infrastructure.adoc[Network & Infrastructure Engineer] — Secure the data plane: segmentation, encryption in transit, mTLS patterns, and traffic-focused observability.

== Environment setup

From the bastion terminal tab, clone this repository and run the lab environment script. This is the attendee setup path: it applies RHACS demo configuration, configures RHACS CLI access, deploys the vulnerable workshop demo applications, and builds/pushes the Quay golden base and frontend images used later in the labs.

[source,bash,subs="attributes",role=execute]
----
git clone https://github.com/mfosterrox/openshift-security-roadshow.git
cd openshift-security-roadshow
bash setup/lab-environment.sh \
  --quay-user {quay_admin_username} \
  --quay-password '{quay_admin_password}'
source ~/.bashrc
----

TIP: Feel free to move on to your first module while the lab environment script runs in the background.
