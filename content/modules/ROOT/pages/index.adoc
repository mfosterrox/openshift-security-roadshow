This course offers hands-on experience with Kubernetes-native security on Red Hat^(R)^ OpenShift Container Platform—covering platform foundations, Advanced Cluster Security (RHACS), and Trusted Software Supply Chain (TSSC). Each module ends with a practical *task* so you apply what you learned before moving on.

== How this Roadshow works

* *This page is home* — Credentials and console links live here. Use the home button anytime you get lost.
* *Navigate with the TOC* — Modules are in the left table of contents, or use *Next* at the bottom of each page.
* *Open consoles in a new tab* — Keep this guide open. On a Mac, use *<cmd> click*; on Windows/Linux, use *<ctrl> click* or right-click → "Open in new tab."
* *Use the bastion* — Run `oc`, `roxctl`, and `podman` on the RHEL bastion (SSH details below).
* *Pick a path* — Tracks are self-contained. Start with the path that matches your audience; each module ends with a hands-on scenario.

== Learning paths

* *101 — Foundations* — OpenShift secure-by-default: RBAC, SCC/PSA, network policy, secrets, image provenance, audit logging, compliance basics, and route TLS.
* *201 — Intermediate* — Hardening and shift-left: custom RBAC/SCC, Vault and External Secrets Operator, CI/CD scanning and signing, audit correlation, compliance tailoring, and admission governance.
* *301 — Advanced* — Enterprise zero trust: platform governance, GitOps compliance pipelines, multi-tenant segmentation, runtime detection and forensics, supply chain integrity, and workload identity.
* *RHACS* — Complete Advanced Cluster Security walkthrough with on realistic examples covering: visibility, vulnerabilities, risk, runtime/deploy/build policy, CI/CD gates, compliance, notifications, API, network security, and installation. 
* *TSSC* — Trusted Software Supply Chain: container and git signing with Trusted Artifact Signer, plus SBOM analysis in Trusted Profile Analyzer.

== Environment setup

From the bastion terminal tab, run the lab environment script. It configures RHACS CLI access (`ROX_CENTRAL_ADDRESS`, `ROX_PASSWORD`, and a generated `ROX_API_TOKEN`), deploys the vulnerable workshop demo applications, and builds/pushes the Quay frontend image used later in the labs.

[source,bash,subs="attributes",role=execute]
----
git clone https://github.com/mfosterrox/openshift-security-roadshow.git
cd openshift-security-roadshow
bash setup/lab-environment.sh \
  --quay-user {quay_admin_username} \
  --quay-password '{quay_admin_password}'
source ~/.bashrc
----
